# 아픈 지점 분석 기본 가이드

채용공고 텍스트에서 다음 항목을 중심으로 회사의 "아픈 지점"을 찾아내세요.

## 기술 부채 신호
- "레거시 시스템 개선", "기술 부채 해소", "마이그레이션" 키워드
- 오래된 기술스택 나열 후 "모던 기술로 전환 예정"

## 스케일 문제
- "트래픽 증가", "대용량 데이터 처리", "안정성 확보" 언급
- "빠르게 성장 중"이면서 인프라 담당자 채용

## 조직·프로세스 문제
- "개발 문화 개선", "프로세스 정립" 표현
- "스타트업이지만 체계를 갖추고 싶다"

## 인력 공백
- 시니어 단독 채용 or 팀 전체를 새로 구성
- "0 to 1" 표현

## 분석 출력 형식
1. **핵심 아픈 지점** (3개 이내, 근거 인용)
2. **제안 방향** (지원자 경력과 연결)
3. **적합도** (0~100%, 근거 1줄)
